Rental workflow

How the flow is framed in this demo

Customer submits service type, location, requested timing, and load details.
Office reviews placement notes, confirms service area, and prepares the quote.
Initial payment covers delivery, first haul, and tax before container placement.
Final tonnage and any extension time can be settled after pickup and weigh-out.

Placement notes

  • It helps if someone is on site at delivery, but the form supports detailed placement instructions if not.
  • Placement surfaces should be selected carefully before delivery because containers are heavy.
  • Overfilled containers or blocked access can delay pickup and create a service follow-up event.
Containers and controls

Quote-based capacity planning

Pricing is left out on purpose for this demo. The site uses a structured request path instead of hardcoded rates, which is smarter until the real pricing model is finalized.

20-yard / 30-yard

Typical build, cleanup, or remodel workflows. Good fit when the office wants flexible delivery and pull timing without forcing fixed pricing on the site.

40-yard commercial

For larger commercial or construction accounts where frequency, tonnage, and haul scheduling matter more than a single simple rental fee.

Swap / pull / hold notes

The form flow supports drop, pull, swap, extension, and relocation so the admin dispatch board can use one intake path instead of email chaos.

Load restrictions

Common rules shown clearly

Do not load above the top rail.
Hazardous material, liquids, and prohibited waste stay out of the container.
Customers should flag heavy material loads so tonnage expectations are clear up front.
